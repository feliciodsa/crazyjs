Reax UI - An explanatory overview

This is a lightweight front-end micro-lib (pure ESM, zero deps) for building UIs with real DOM nodes. Components are just functions that return native elements, rendering is explicit, reactivity is opt-in, and routing is minimal, including support for HTML pages loaded via fe



Reax UI - Filosofia

  1. • No virtual DOM — you work with the real DOM.
  2. • Reactivity only when needed via Reactive.
  3. • Minimal structure — everything as ESM; no build required.
  4. • Simple Router — hash/history, smooth transitions, skeleton loading, prefetch, and HTML page loading
