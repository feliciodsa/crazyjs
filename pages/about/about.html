Reax UI - Visão geral explicativa

É uma micro-lib front-end (ESM, zero dependências) para montar interfaces usando nós DOM reais. A ideia é ser direta: componentes são funções que devolvem elementos nativos, o render é explícito, a reatividade é opcional e o roteamento é leve, com suporte a páginas HTML carregadas por fetch.



Reax UI - Filosofia

  1. • Sem virtual DOM: você lida com o DOM real.
  2. • Reatividade só quando precisa (Reactive).
  3. • Estrutura mínima: tudo em ESM, sem build obrigatório.
  4. • Router simples: hash/history, transições suaves, loading com skeleton, prefetch e opção de carregar HTML externo.
